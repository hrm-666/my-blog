## 前言

这篇笔记主要整理我对 Q-learning 的第一轮理解。相比直接记公式，我更想先弄清楚几个问题：

- `Q(s, a)` 到底表示什么？
- `reward` 和 `Q` 值有什么区别？
- 为什么更新公式里一定要看 `next_state`？
- 为什么 Q 表的价值会有一种“从终点往前传”的感觉？
- `alpha`、`gamma`、`epsilon` 分别在控制什么？

下面的理解主要结合表格型 Q-learning 和 `FrozenLake` 环境展开。

## Q-learning 在学什么

Q-learning 的核心目标不是直接学习“哪个状态好”，而是学习：

> 在状态 `s` 下，如果我采取动作 `a`，从这一步开始，未来大概能获得多少累计回报。

这个量叫作动作价值函数，也就是：

```text
Q(s, a)
```

所以 `Q(s, a)` 不是当前这一步的即时奖励，而是一个对未来累计收益的估计。更具体一点说，它表示：

> 当前先做动作 `a`，之后继续尽量做出较优决策，最终这条路径大概值多少。

因此，Q-learning 学到的不是单纯的状态价值，而是“状态-动作”这一对组合的价值。

## reward 和 Q 值的区别

这两个概念很容易混。

`reward` 是当前这一步执行动作之后，环境立刻给出的反馈。  
`Q(s, a)` 是从当前状态采取当前动作之后，对未来累计回报的估计。

可以粗略理解为：

- `reward` 只看眼前；
- `Q(s, a)` 看的是从这一步出发，后面整条路大概值多少钱。

以 `FrozenLake` 为例，大多数普通格子的即时奖励都是 `0`。但是有些动作的 Q 值仍然可能比较高，因为这个动作虽然当下没有拿到奖励，却可能把智能体带到一个离终点更近的位置。也就是说：

> 当前这一步没有奖励，不代表这一步没有价值。

这也是强化学习里“长期收益”比“即时反馈”更重要的地方。

## Q 表是什么

在最基础的表格型 Q-learning 中，我们会维护一张 Q 表。

这张表可以这样理解：

- 行表示状态 `state`；
- 列表示动作 `action`；
- 每个格子存储一个 `Q(state, action)`。

例如在 `FrozenLake` 代码中可能会写：

```python
state_size = env.observation_space.n
action_size = env.action_space.n
Q = np.zeros((state_size, action_size))
```

这说明每个状态都有若干个可选动作，而每个“状态-动作”组合都有一个对应的估计价值。训练的过程，本质上就是不断和环境交互，然后把这张 Q 表更新得越来越靠谱。

## Q-learning 的更新公式

Q-learning 最核心的更新公式是：

```text
Q(s, a) = Q(s, a) + alpha * (reward + gamma * max Q(s', a') - Q(s, a))
```

也可以拆成更容易理解的形式：

```text
新值 = 旧值 + 学习率 * (目标值 - 旧值)
```

其中目标值是：

```text
target = reward + gamma * max Q(s', a')
```

这几个量分别表示：

- `Q(s, a)`：当前这对状态和动作原来的估计；
- `reward`：当前动作带来的即时奖励；
- `gamma`：折扣因子，决定未来奖励有多重要；
- `max Q(s', a')`：到达下一状态后，未来能获得的最好收益估计；
- `alpha`：学习率，决定这次新经验对旧估计的影响有多大。

所以这条公式想表达的是：

> 当前动作的价值，应该逐渐向“当前奖励 + 下一状态的最优未来价值”靠近。

它不是一次性把旧值完全推翻，而是用 `alpha` 控制更新幅度，让 Q 值逐步逼近更合理的估计。

## 为什么一定要看 next_state

这是理解 Q-learning 的关键之一。

当智能体在状态 `s` 执行动作 `a` 后，真正发生的是：

1. 环境给出一个即时奖励 `reward`；
2. 智能体进入下一个状态 `s'`；
3. 从 `s'` 开始，后面还会继续产生未来收益。

所以当前这一步的价值，不可能只由当前状态决定。它由两部分组成：

- 眼前拿到多少奖励；
- 到达下一个状态后，未来还能获得多少收益。

因此更新时必须看：

```text
max Q(s', a')
```

而不是看当前状态中其他动作的 Q 值。因为当前动作已经执行完了，未来能怎么走，取决于你已经到达的下一个状态。

## 为什么价值像是从后往前传递

刚开始 Q 表通常全是 `0`，所以智能体并不知道哪条路更好。

只有当智能体真的接近终点，甚至到达终点时，价值信息才第一次出现。例如在 `FrozenLake` 中，如果某一步到达终点并得到 `reward = 1`，那么终点前一个状态中通向终点的动作 Q 值会先升高。

下一次训练中，当智能体访问更前面的状态时，它就能看到：

```text
后面的某个状态已经更值钱了
```

于是这个价值会继续向更早的状态传播。

所以智能体在环境里是一步一步向前走的，但 Q 值的改善看起来像是从终点附近开始，一层层往前倒推。这就是 Q-learning 里非常重要的“价值传播”直觉。

## 三个重要参数

### alpha：学习率

`alpha` 控制新经验对旧估计的影响有多大。

- `alpha` 太大：更新更激进，学得快，但可能震荡；
- `alpha` 太小：更新更保守，更平滑，但学习速度慢。

我之前容易把它理解成“判断这次更新有多重要”。更准确地说，`alpha` 不是判断这次更新对不对，而是控制这次新信息在更新中占多大权重。

### gamma：折扣因子

`gamma` 控制智能体有多看重未来。

- `gamma = 0`：只看眼前奖励；
- `gamma` 接近 `1`：非常看重长期收益。

需要注意的是，`gamma = 0` 不等于“没学”。它仍然在学习，只是学到的是一种只关心即时奖励的短视策略。

在 `FrozenLake` 这种终点奖励比较晚才出现的任务里，如果 `gamma = 0`，前面大多数普通步骤的即时奖励都是 `0`，智能体就很难学到通向终点的长期策略。

### epsilon：探索概率

`epsilon` 控制探索概率。

在 `epsilon-greedy` 策略中：

- 以 `epsilon` 的概率随机选择动作；
- 否则选择当前 Q 表中价值最高的动作。

它解决的问题是：如果一开始就完全贪心，智能体可能只会反复走少数几条路径，很多状态和动作根本没有机会被尝试到。

不过也要纠正一个说法：“不探索就完全不会更新”并不准确。即使不探索，只要智能体在执行动作，仍然会发生更新。问题在于，这些更新很可能局限在很窄的一部分路径上，容易陷入次优策略。

## 为什么 FrozenLake 适合入门

我现在用的是：

```python
env = gym.make("FrozenLake-v1", is_slippery=False)
```

这里的 `is_slippery=False` 不只是“新手友好”，它还有一个很重要的含义：环境变成了确定性的。

也就是说，在同一个状态执行同一个动作，结果更可预测。这会让 Q-learning 更容易收敛，也更方便观察 Q 值是如何传播的。

如果改成 `is_slippery=True`，环境就会带有随机性。例如你想往右走，实际可能滑向别的方向。这样同一个 `(state, action)` 的后果就不再固定，训练会更难，Q 值也需要更多回合才能稳定下来。

## 终止状态怎么处理

“到终点就不更新了”这个说法不完全准确。

更准确地说：

> 到达终止状态的那一步，通常仍然要更新当前这一步的 Q 值，只是终止之后已经没有后续动作，所以未来价值应该视为 `0`。

也就是说，当 `next_state` 是终止状态时：

```text
target = reward
```

更严谨的代码可以写成：

```python
if done:
    target = reward
else:
    target = reward + gamma * np.max(Q[next_state])
```

如果直接写成：

```python
target = reward + gamma * np.max(Q[next_state])
```

在 `FrozenLake` 里通常也不一定会出大问题，因为终止状态那一行 Q 值往往本来就是 `0`。但从概念上看，显式区分 `done` 会更清楚。

## 训练和推理的区别

训练和推理本质上是在做两件不同的事情。

训练阶段：

- 智能体需要探索；
- 每一步都要根据新经验更新 Q 表；
- 目标是让 Q 表越来越准确。

推理阶段：

- 不再更新 Q 表；
- 通常直接选择 `argmax(Q[state])`；
- 目标是使用已经学好的策略完成任务。

所以如果把代码拆成 `FrozenLake.py` 和 `FrozenLake_inference.py`，前者负责训练，后者负责使用训练好的 Q 表，这个结构是比较清楚的。

## Q-learning 和监督学习的区别

监督学习通常有一个明确的数据集，数据集中已经给好了标准答案。模型要做的是尽量拟合这些标签。

而 Q-learning 中没有人直接告诉智能体“标准 Q 值是多少”。它只能通过和环境交互获得：

- 当前状态；
- 执行动作；
- 即时奖励；
- 下一状态。

然后用：

```text
reward + gamma * max Q(next_state, :)
```

构造出当前这一步的学习目标。

所以 Q-learning 的目标值不是人工标注出来的，而是由环境反馈和当前 Q 表共同构造出来的。这也是强化学习和监督学习很不一样的地方。

## 对应到代码流程

一次训练回合大致可以理解为：

1. 重置环境，得到初始状态；
2. 用 `epsilon-greedy` 选择动作；
3. 执行动作，得到 `next_state` 和 `reward`；
4. 用 Q-learning 更新公式修改 `Q[state, action]`；
5. 转移到下一个状态继续；
6. 回合结束后衰减 `epsilon`。

对应到核心代码就是：

```python
if np.random.rand() < epsilon:
    action = env.action_space.sample()
else:
    action = greedy_action(Q, state)

next_state, reward, terminated, truncated, info = env.step(action)
done = terminated or truncated

old_value = Q[state, action]

if done:
    target = reward
else:
    target = reward + gamma * np.max(Q[next_state])

Q[state, action] = old_value + alpha * (target - old_value)
```

这个过程可以概括成：

> 先试一个动作，再观察结果，然后用这次结果修正当前动作的价值估计。

## 目前最该记住的几句话

- `Q(s, a)` 不是当前奖励，而是从当前动作出发后的未来累计回报估计。
- Q-learning 学的是动作价值，不只是状态价值。
- 当前 Q 值更新要看 `next_state`，因为未来收益从下一状态开始决定。
- 价值信息通常从终点附近开始，逐步向前传播。
- `alpha` 控制更新幅度，`gamma` 控制看多远，`epsilon` 控制探索多少。
- 训练是为了学习 Q 表，推理是为了使用 Q 表。

## 容易记错的点

### 误区一：gamma = 0 相当于没学

不对。`gamma = 0` 仍然在学，只是只学习即时奖励。

### 误区二：不探索就完全不会更新

不对。不探索仍然会更新，但很容易只在有限路径上更新，学不到真正好的策略。

### 误区三：到终点就不更新了

不对。到达终点的那一步仍然要更新，只是后续未来价值通常视为 `0`。

### 误区四：Q 初值大小没影响

不完全对。Q 初值会影响前期学习速度和探索倾向，只是未必影响最终最优策略。
